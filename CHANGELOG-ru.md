# Журнал изменений

Все заметные изменения проекта будут документироваться в этом файле.

## [1.3.0] - 2026-06-29

### Добавлено

- Обнаружение Claude subagents в `abtop-py` из локальных файлов metadata и transcript.
- Отображение `Model/Rsn` для Codex reasoning effort, если он доступен в rollout `turn_context`.

### Изменено

- Верхний график скорости токенов теперь использует тот же rolling-ряд, что и числовая метка `Token Rate`, чтобы пачечные обновления Codex token-count не выглядели как одиночные per-tick пики.
- Панель `context` теперь имеет фиксированную область содержимого в шесть строк: пять строк Unicode block-level графика токенов и одну строку временной оси; строки проектов обрезаются по доступному месту.
- График скорости токенов использует вертикальный масштаб `32k/min` по умолчанию и расширяется, когда видимая скорость токенов превышает это значение; текущий масштаб показывается под `Token Rate`.

### Документация

- README-файлы теперь описывают Claude-only структурированные subagents, отображение Codex reasoning effort, отсутствие структурированного поля Codex speed и различие между rolling-графиком token-rate и событийной историей `tokens/turn`.

## [1.2.0] - 2026-06-22

### Добавлено

- Временные привязки в UI: колонка `Last` в таблице сессий, датированные заголовки `CHAT` и `TIMELINE`, формат дат с названием месяца.
- Статичная относительная шкала времени под графиком скорости токенов: подписанный левый край и деления по 30 секунд к правому краю `now`.
- Панель активности проектов: одна строка на проект, сортировка по последней активности, агрегированное состояние, возраст последнего события, маркеры числа сессий и скорость токенов по проекту.
- Явная обработка Codex approval: tool calls с escalated sandbox request определяются как ожидание решения, отображаются как `Decide` / `Approve` и группируются при перекрывающихся ожиданиях.
- Строки `Queued` в timeline для команд, которые были записаны во время ожидания approval, но еще не выполнялись.

### Изменено

- Верхняя строка показывает неизвестную атрибутированную память агентов как `Σ-`, а не вводящее в заблуждение `Σ0M`.
- Возраст данных quota отображается как `upd Ns`, а не как голое `Ns ago`.
- Подсказка настройки Claude quota теперь указывает на `./abtop-py.py --setup`.
- Числовая скорость токенов считается по rolling window.
- Завершенные Codex tool calls используют явный `Wall time` из вывода инструмента, если он доступен, чтобы время ожидания approval не считалось временем выполнения команды.
- Средний ряд панелей стал на одну строку выше, чтобы подписи reset в quota не накладывались на footer скорости токенов.

### Документация

- README-файлы описывают метрики верхней строки, временную ось графика токенов, строки активности проектов, датированные заголовки chat/timeline, обработку Codex approval и ограничения атрибуции памяти.
- PlantUML-диаграммы обновлены: добавлены host metrics, временные метаданные, агрегация активности проектов, approval/queued timeline и новая модель UI-панелей.

## [1.1.0] - 2026-06-19

### Добавлено

- Верхняя btop-style строка статуса в `abtop-py` с CPU, памятью, load average, суммарной памятью агентов, средним использованием контекста, временем и счетчиками сессий.
- Метка ожидания Codex между шагами, чтобы не показывать `waiting for input` во время коротких idle-пауз после сообщений ассистента.

## [1.0.0] - 2026-06-18

### Добавлено

- Первый публичный релиз `abtop-py`.
- Однофайловый Python-монитор без внешних зависимостей для сессий Claude Code и Codex CLI/Desktop.
- Интерактивный curses-дашборд с панелями контекста, квот, токенов, проектов, портов, MCP-серверов, сессий, чата, subagents и timeline.
- Одноразовый текстовый снимок через `--once`.
- JSON-снимок через `--json`.
- Установка Claude StatusLine helper через `--setup`.
- Полностью локальный сбор данных из состояния процессов, файлов сессий, транскриптов, rollout-логов, состояния Git и файлов кэша лимитов.
- Различение ожидания обычного ввода пользователя и явного решения или подтверждения.
- README-файлы на английском и русском языках.
- Архитектурные диаграммы PlantUML на английском и русском языках.
