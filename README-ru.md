# abtop-py

[![Version](https://img.shields.io/badge/version-1.0.0-blue.svg)](CHANGELOG-ru.md)

`abtop-py` - это терминальный монитор локальных сессий AI-агентов в одном Python-файле без внешних зависимостей. Он отслеживает активность Claude Code и Codex CLI/Desktop, читая локальное состояние процессов, файлы сессий, транскрипты и кэши лимитов.

Программа не вызывает API провайдеров и не требует сторонних Python-пакетов.

## Возможности

- Живой curses-дашборд для сессий Claude Code и Codex.
- Одноразовый текстовый вывод для скриптов и терминалов без TTY.
- JSON-снимок для интеграций.
- Итоги по токенам, использование контекста, история скорости токенов и временные шкалы сессий.
- Видимость вызовов инструментов и дочерних процессов, включая память и открытые порты.
- Панель MCP-серверов с parent/profile/activity, если процессы MCP видны системе.
- Отдельные статусы ожидания для обычного ввода пользователя и явного решения/подтверждения.
- Ветка Git и счетчики измененных/добавленных файлов для активных проектов.
- Отображение лимитов Claude и Codex, если локальные данные доступны.

## Требования

- Python 3.8 или новее.
- Linux дает наиболее полные данные о процессах и портах через `/proc`.
- На других Unix-подобных системах возможен резервный сбор через `ps` и `lsof`, если они доступны.

## Использование

Запустить интерактивный дашборд:

```bash
./abtop-py.py
```

Вывести один текстовый снимок и завершить работу:

```bash
./abtop-py.py --once
```

Вывести один JSON-снимок и завершить работу:

```bash
./abtop-py.py --json
```

Установить Claude StatusLine helper для сбора лимитов Claude:

```bash
./abtop-py.py --setup
```

## Опции

```text
--once                         вывести один текстовый снимок и выйти
--json                         вывести один JSON-снимок и выйти
--interval SECONDS             интервал обновления, по умолчанию 2.0
--hide claude|codex            скрыть тип агента, можно повторять
--claude-config-dir PATH       добавить корень конфигурации Claude
--setup                        установить Claude StatusLine helper для лимитов
--version                      вывести версию и выйти
```

## Конфигурация

Необязательная конфигурация читается из:

```text
~/.config/abtop/config.toml
```

Поддерживаемые простые массивы:

```toml
hidden_agents = ["codex"]
claude_config_dirs = ["~/.claude-work"]
```

`hidden_agents` принимает значения `claude` и `codex`. `claude_config_dirs` добавляет дополнительные корни конфигурации Claude Code для сканирования.

## Источники данных

`abtop-py` читает только локальные данные:

- Метаданные сессий Claude Code из `.claude*/sessions/*.json`.
- Транскрипты Claude Code из `.claude*/projects/**/*.jsonl`.
- Rollout-логи Codex из `.codex/sessions/**/rollout-*.jsonl`.
- Данные о процессах, памяти, дочерних процессах и портах из операционной системы.
- Подсказки о MCP-серверах из видимых командных строк дочерних процессов.
- Состояние Git из рабочих директорий сессий.

## Ограничения

- Показываются только сессии, найденные в локальных файлах или среди запущенных процессов.
- Часть данных о процессах может быть недоступна без `/proc`, `ps` или `lsof`.
- Данные о лимитах зависят от локального вывода Claude StatusLine или недавних событий Codex rollout.
