# abtop-py

[![Version](https://img.shields.io/badge/version-1.3.0-blue.svg)](CHANGELOG-ru.md)

`abtop-py` - это терминальный монитор локальных сессий AI-агентов в одном Python-файле без внешних зависимостей. Он отслеживает активность Claude Code и Codex CLI/Desktop, читая локальное состояние процессов, файлы сессий, транскрипты и кэши лимитов.

Программа не вызывает API провайдеров и не требует сторонних Python-пакетов.

## Возможности

- Живой curses-дашборд для сессий Claude Code и Codex.
- Верхняя btop-style строка статуса с CPU, памятью, load average за одну минуту, атрибутированной памятью агентов, контекстом и счетчиками сессий.
- Одноразовый текстовый вывод для скриптов и терминалов без TTY.
- JSON-снимок для интеграций.
- Итоги по токенам, использование контекста, история скорости токенов с относительной временной шкалой и датированные timeline сессий.
- Видимость вызовов инструментов и дочерних процессов, включая память и открытые порты.
- Панель MCP-серверов с parent/profile/activity, если процессы MCP видны системе.
- Отображение `Model/Rsn` для Codex-сессий, если reasoning effort есть в rollout `turn_context`.
- Панель Claude subagents из локальных `subagents/*.meta.json` и соответствующих транскриптов subagent.
- Отдельные статусы ожидания для обычного ввода пользователя, пауз между шагами и явного решения/подтверждения.
- Панель активности проектов: одна строка на проект, сортировка по последней активности, состояние, возраст последнего события и скорость токенов.
- Временные привязки в таблице сессий, заголовке чата и заголовке timeline.
- Ожидания Codex approval помечаются как решения, перекрывающиеся approval-ожидания схлопываются, queued-команды не отображаются как долго выполняющаяся работа.
- Отображение лимитов Claude и Codex, если локальные данные доступны.

## Заметки по дашборду

Верхняя строка использует компактные метки:

- `CPU` / `MEM` - загрузка хоста.
- `L` - load average за одну минуту.
- `agents Σ...` - память, которую удалось надежно привязать к видимым деревьям процессов агентов. `Σ-` означает, что память неизвестна, а не равна нулю.
- `ctx%...` - среднее ненулевое заполнение context window по видимым сессиям.

Панель `context` имеет фиксированную область содержимого в шесть строк: пять строк для rolling-графика скорости токенов и одну строку для относительной временной оси. График использует уровни Unicode-блоков (`▁▂▃▄▅▆▇█`), поэтому столбцы выглядят плавнее, и масштабируется минимум до `32k/min`, расширяясь при превышении видимой скоростью токенов этого потолка. Текущий масштаб графика показывается под `Token Rate`. Число `Token Rate` и график используют одно и то же rolling window, поэтому пачечные события Codex token-count сглаживаются, а не выглядят как одиночные пики. Нижняя ось относительна: правый край - `now`, деления идут через 30 секунд, а левый край подписывается фактическим возрастом даже если он не кратен 30 секундам. В правой части показываются только те строки проектов, которые помещаются в фиксированную панель.

Колонка таблицы сессий `Model/Rsn` показывает модель и Codex reasoning effort, если он доступен, например `gpt-5.5/high`. У Claude-сессий reasoning effort обычно отсутствует, поэтому показывается только модель.

Детальная панель `SUBAGENTS` заполняется из файлов Claude Code subagent внутри каталога проекта сессии. Rollout-логи Codex сейчас не дают структурированного списка subagents, поэтому Codex-сессии обычно показывают там `none`, даже если внутри Codex используется multi-agent механика.

Нижняя панель `tokens` использует историю `tokens/turn`. Этот график событийный: Codex добавляет точки при поступлении rollout-событий `token_count`, поэтому он может оставаться ступенчатым или разреженным, даже когда верхний график скорости сглажен.

Панель `projects` теперь показывает активность, а не Git-сводку:

```text
Project      State   Last   Tok/m
vaultui-py   Work      5s   13.0k
```

Строки сортируются по `Last`, самые свежие сверху. `Tok/m` - скорость активных токенов по проекту за текущий refresh interval, приведенная к токенам в минуту.

Заголовки `CHAT` и `TIMELINE` содержат компактные даты вида `Jun 22 00:04`. Строки timeline по-прежнему показывают длительности, а заголовок показывает общий диапазон времени.

## Требования

- Python 3.8 или новее.
- Linux дает наиболее полные данные о процессах и портах через `/proc`.
- На других Unix-подобных системах возможен резервный сбор через `ps` и `lsof`, если они доступны.

## Использование

Запустить интерактивный дашборд:

```bash
./abtop-py.py
```

Вывести один текстовый снимок и завершить работу:

```bash
./abtop-py.py --once
```

Вывести один JSON-снимок и завершить работу:

```bash
./abtop-py.py --json
```

Установить Claude StatusLine helper для сбора лимитов Claude:

```bash
./abtop-py.py --setup
```

Команда создает `~/.claude/abtop-statusline.sh` и прописывает Claude Code `statusLine`, чтобы локальные данные quota Claude попадали в `abtop-rate-limits.json`.

## Опции

```text
--once                         вывести один текстовый снимок и выйти
--json                         вывести один JSON-снимок и выйти
--interval SECONDS             интервал обновления, по умолчанию 2.0
--hide claude|codex            скрыть тип агента, можно повторять
--claude-config-dir PATH       добавить корень конфигурации Claude
--setup                        установить Claude StatusLine helper для лимитов
--version                      вывести версию и выйти
```

## Конфигурация

Необязательная конфигурация читается из:

```text
~/.config/abtop/config.toml
```

Поддерживаемые простые массивы:

```toml
hidden_agents = ["codex"]
claude_config_dirs = ["~/.claude-work"]
```

`hidden_agents` принимает значения `claude` и `codex`. `claude_config_dirs` добавляет дополнительные корни конфигурации Claude Code для сканирования.

## Источники данных

`abtop-py` читает только локальные данные:

- Метаданные сессий Claude Code из `.claude*/sessions/*.json`.
- Транскрипты Claude Code из `.claude*/projects/**/*.jsonl`.
- Метаданные и транскрипты Claude Code subagents из `.claude*/projects/<project>/<session>/subagents/`.
- Rollout-логи Codex из `.codex/sessions/**/rollout-*.jsonl`.
- Данные о процессах, памяти, дочерних процессах и портах из операционной системы.
- Подсказки о MCP-серверах из видимых командных строк дочерних процессов.
- Состояние Git из рабочих директорий сессий.
- CPU, память и load average хоста из `/proc` на Linux.

## Ограничения

- Показываются только сессии, найденные в локальных файлах или среди запущенных процессов.
- Часть данных о процессах может быть недоступна без `/proc`, `ps` или `lsof`.
- Для Codex Desktop/recent rollout сессий может не быть надежной привязки к owning process tree; память таких сессий отображается как неизвестная (`-` / `Σ-`), а не как нулевая.
- Codex reasoning effort показывается, если есть `turn_context.payload.effort` или `collaboration_mode.settings.reasoning_effort`. Отдельная настройка скорости Codex не показывается, потому что текущие rollout-логи не дают ее как стабильное структурированное поле.
- Codex-сессии сейчас не предоставляют структурированные записи subagents; панель subagents является Claude-специфичной, кроме fallback на tool calls с явно subagent-подобным именем.
- Данные о лимитах зависят от локального вывода Claude StatusLine или недавних событий Codex rollout.
